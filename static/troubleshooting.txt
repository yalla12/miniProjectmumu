★홈화면:이동건
부트스트랩을 이용해서 카드들은 쉽게 적용하였으나 jinja2라는 새롭게 사용해본 템플릿엔진의 사용이 익숙하지 않고
이전에 ajax를 이용한 스크립트 작성과는 달라서 처음 생각했던것보다 더 까다로웠고 오래걸렸습니다.


1.토글시 이미지 변경문제
문제점:document.getElementById("club").src = ""; 를 이용해서 해당 아이디값을 가진 친구의 이미지 src 항목을 스태틱
폴더에 저장해둔 그림으로 일괄 변경 시키게 했더니 43장의 그림중에 맨첫번째 1장만 바뀌는 것

해결방법:$("[id='man']").attr("src", "static/man.jpg");를 사용

2.다시 토글시 원래 이미지 복원
문제점:토글 이벤트는 스크립트 항목에서 이루어지는데 jinja2가 스크립트와 어떻게 긴밀히 연결시켜야할지 모르겠음

해결방법:이미지 변경을 2번하는데 마지막번째는 다시 for문을 돌리는 비슷한 사례를 찾지못하여 그냥 새로고침으로 마무리지음.






제가 구현한 기능중에 토글시 이미지를 변경시키는 기능이 있는데 
document.getElementById("club").src = ""; 를 이용해서 해당 아이디값을 가진 친구의 이미지 src 항목을 스태틱
폴더에 저장해둔 그림으로 일괄 변경 시키게 했더니 43장의 그림중에 맨첫번째 1장만 바뀌는 것이었습니다.
화요일에 하루종일 해당 기능을 풀어내기위해서 강의랑 구글링을 했고 그 다음날인 수요일에 문제점을 해결했습니다.
문제점이 뭐였냐면 하나의 id 값으로 여러 액션을 적용할 수 없다는 개념을 몰랐기 때문이었습니다. 보통은 이렇게 여러개를
 조작하려면 id 보다는 class 값을 사용하는것이 좋다고 구글링을 통해 배웠습니다.
jquery를 이용하면 이미지를 일괄 변경 할 수 있다는걸 알게 되었는데 처음엔 class 값을 넣었다가 혹시 id도 될까해서 해봤는데
잘되길래 현재는 id 값으로 적용되어있습니다. 왜 jquery를 쓰면 id 값으로 하는데도 모두 적용되는지는 알 수 없었습니다.
$("[id='man']").attr("src", "static/man.jpg");를 사용했습니다.

두번째 토글 클릭시에 원래 이미지를 되찾게 하는것에 문제점이 아주 어려웠습니다.
html 쪽에서 jinja 문법으로 for 문을 돌려서 카드에 db의 이미지들을 나타냈는데 
토글시에는 함수를 호출해서 스크립트에서 조작을 하게 만들었기 때문에 도데체 스크립트 코드를 어떻게 해야하는지 모르겠습니다.
for 문을 돌려서 여러개의 이미지를 적용시킨것을 다시 원래 상태로 만드는 방법을 찾아야하는데 수요일 새벽까지 찾아도
비슷한 경우에 해당하는것을 구글링 할 수 없었습니다. 스크립트에서 이번엔 jquery로 스크립트를 돌려서 다시 적용시키게 해야하나
했는데 서버에 요청해서 받아온 db데이터는 json 형태도 아니고 리스트로 전부 받아온거라 적용하는것도 미숙하고 적용할 html의
div 항목들에는 jinja 문법으로 범벅이 되어있어서 스크립트에서 해당 jinja 변수를 읽지를 못하는 둥 결국 포기하게되었습니다.
